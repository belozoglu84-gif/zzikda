$ARGUMENTS 브랜드의 프랜차이즈 가맹모집 랜딩페이지를 만든다.

## 반드시 아래 5단계를 순서대로 진행할 것. 단계를 건너뛰지 마라.

---

### 1단계: 브랜드 분석

사용자에게 다음 정보를 질문한다 (모르는 항목은 "나중에"로 넘길 수 있음):

**필수 정보:**
- 브랜드명, 업종 (치킨/분식/카페/한식/고기 등)
- 대표 메뉴 3~6개
- 핵심 차별점 3가지
- 전화번호, 주소
- 브랜드 사진이 있는지 (있다면 images/ 폴더에 넣어달라고 안내)

**있으면 좋은 정보:**
- 매장 수, 운영 기간, 핵심 수치 (매출, 리뷰수 등)
- 브랜드 컬러 (없으면 landing-design 스킬의 업종별 추천 컬러 제안)
- 가맹비/보증금/인테리어 비용 (공개 가능 여부)
- SNS, 배달앱 링크

답변을 `기획서_$ARGUMENTS_랜딩페이지.md`에 정리한다.

**1단계 완료 후 사용자에게 확인받고 2단계로 진행한다.**

---

### 2단계: 레퍼런스 선택

1. CLAUDE.md의 "리소스 경로"에서 레퍼런스 폴더 경로를 읽는다
2. 해당 경로의 `종합분석.md`를 읽어서 전체 트렌드를 파악한다
3. 레퍼런스 폴더들을 스캔해서 유사 업종의 report.md를 읽는다
4. 3~5개 레퍼런스를 추천하고, 각각의 강점을 한 줄로 설명한다
5. 사용자와 함께 "이 부분은 A브랜드처럼, 저 부분은 B브랜드처럼" 결정한다

> 레퍼런스 폴더가 없거나 비어있으면, landing-sections 스킬만으로 진행한다.

**2단계 완료 후 사용자에게 확인받고 3단계로 진행한다.**

---

### 3단계: 설계

1. landing-sections 스킬을 읽고 업종에 맞는 섹션 조합을 제안한다
2. 사용자와 함께 최종 섹션 구성을 확정한다
3. landing-design 스킬을 읽고 디자인 시스템을 확정한다:
   - 컬러 시스템 (배경, 텍스트, 액센트)
   - 폰트 조합 (영문 헤드라인 + 한글 본문)
   - 톤앤매너
4. 기획서에 확정된 섹션 구성 + 디자인 시스템을 추가한다

**3단계 완료 후 사용자에게 확인받고 4단계로 진행한다.**

---

### 4단계: 코딩

1. landing-dev 스킬과 landing-design 스킬을 반드시 읽은 후 코딩을 시작한다
2. `index.html` 파일을 생성한다
3. 섹션별로 순서대로 구현한다:
   - 먼저 `<head>` (메타태그, Tailwind, 폰트, 커스텀 CSS)
   - 네비게이션 바
   - 히어로 섹션
   - (확정된 섹션 순서대로)
   - 푸터
   - `<script>` (인터랙션 JS)
4. 각 섹션 완료 후 사용자에게 확인받는다
5. 이미지가 아직 없는 경우 placeholder를 사용하고 주석으로 표시한다

---

### 5단계: 검증

아래 체크리스트를 확인한다:

- [ ] 모바일 반응형: Tailwind 반응형 클래스 (sm:/md:/lg:) 적용
- [ ] 이미지: 모든 `<img>`에 alt 텍스트, `<picture>` + WebP 사용
- [ ] CTA: 최소 3개 이상, 네비/가맹안내/문의 섹션에 배치
- [ ] 플로팅 CTA: 모바일 하단 고정 버튼 있는지
- [ ] SEO: `<title>`, `<meta description>`, Open Graph 태그
- [ ] 접근성: 시맨틱 태그, `lang="ko"`, `prefers-reduced-motion`
- [ ] 성능: 히어로 이미지 preload, 나머지 lazy loading
- [ ] 전화번호: `tel:` 링크로 클릭 가능

결과를 사용자에게 보고한다.
