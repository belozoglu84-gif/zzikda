# 레퍼런스 기반 랜딩페이지 제작

## 핵심 원칙

규칙 900줄을 외우는 것보다, 잘 만든 레퍼런스 2개를 참고하는 게 낫다.
기획서를 받으면 **레퍼런스 2개를 먼저 골라라**. 나머지 스킬(design/dev/sections)은 보조 규칙이다.

---

## 워크플로우 (반드시 이 순서)

```
1. 작업폴더에 brief.md 생성 (brief-template.md에서 복사)
2. 사용자에게 [미입력] 항목 질문해서 기획서 채우기
3. 아래 카탈로그에서 레퍼런스 2개 선택 → brief.md에 기록
4. 선택된 레퍼런스의 blueprint.json 로딩
5. blueprint 슬롯에 brief.md 값 매핑 (landing-mapping 스킬)
6. landing-dev 스킬의 코드 패턴으로 index.html 생성
7. images/ 폴더의 사용자 자료를 코드에 연결
```

### 관련 파일
- `brief-template.md` — 기획서 양식 (이 폴더에 있음)
- `blueprint-schema.md` — 블루프린트 구조 설명 (이 폴더에 있음)
- `blueprints/` — 각 레퍼런스의 blueprint.json (레퍼런스 폴더에 있음)

## 레퍼런스 파일 활용법

선택한 레퍼런스의 폴더에서 이 파일들을 읽는다:

| 파일 | 무엇을 가져올까 |
|------|---------------|
| `report.md` | **가장 먼저 읽기** — 섹션구성, CTA, 색상, 폰트 요약 |
| `styles.json` | 색상 HEX값, 폰트 크기, 여백 수치를 그대로 참고 |
| `meta.json` | CTA 버튼 텍스트, 네비 구조, 기술 스택 |
| `animations.json` | transition 속도, easing, 슬라이더 설정 |
| `desktop_full.png` | 전체 레이아웃 시각 참고 |
| `mobile_full.png` | 모바일 레이아웃 시각 참고 |
| `sections/` | 개별 섹션 디자인 참고 |

경로: CLAUDE.md의 `리소스 경로` 섹션에서 확인하거나, `references/{번호}_{브랜드명}/` 패턴으로 접근.
