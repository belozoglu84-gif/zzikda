# 블루프린트 + 기획서 매핑 워크플로우

## 이 스킬의 역할

레퍼런스 블루프린트(디자인 틀)에 사용자 기획서(실제 내용)를 합쳐서
최종 랜딩페이지를 만드는 **전체 과정**을 관리한다.

---

## 전체 워크플로우

```
[Phase 1] 기획서 수집
    ↓
[Phase 2] 레퍼런스 선택
    ↓
[Phase 3] 블루프린트 매핑
    ↓
[Phase 4] 코드 생성
    ↓
[Phase 5] 사용자 자료 적용
```

---

## Phase 1: 기획서 수집

### 작업 폴더에 brief.md 생성

1. 하네스의 `brief-template.md`를 작업 폴더에 `brief.md`로 복사
2. 사용자에게 `[미입력]` 항목을 순서대로 물어본다
3. 사용자가 이미 제공한 정보가 있으면 자동으로 채운다

### 질문 순서 (중요도순)

```
1차 (필수 — 이것만 있어도 시작 가능):
  - 브랜드명
  - 업종
  - 원하는 분위기

2차 (구조 결정에 필요):
  - 메뉴 3~6개 (이름 + 한줄 설명)
  - 가맹 비용 항목
  - 대표 전화번호

3차 (완성도 높이기):
  - 슬로건
  - 본사 지원 항목
  - 성공 사례
  - 회사 정보 (푸터)

4차 (자료 연결):
  - 준비된 이미지 파일 목록 (images/ 폴더 스캔)
  - 브랜드 컬러 (있으면)
  - 추가 요구사항
```

### 이미지 자동 감지

작업 폴더의 `images/` 폴더를 스캔해서:
- 파일 목록을 brief.md의 "준비된 자료"에 자동 기입
- 파일명으로 용도 추측: hero, menu, interior, logo 등의 키워드 확인
- 추측이 안 되면 사용자에게 "이 사진은 어디에 쓸까요?" 질문

## Phase 2: 레퍼런스 선택

### landing-reference 스킬의 3단계 선택 수행

1. brief.md의 `업종` → STEP 1 후보 뽑기
2. brief.md의 `원하는 분위기` → STEP 2 좁히기
3. brief.md의 메뉴/가맹/CTA 요구사항 → STEP 3 최종 2개

### 선택 결과를 brief.md에 기록

```markdown
## 11. 선택된 레퍼런스
| 항목 | 값 |
|------|-----|
| 레퍼런스 1 | 003 하남돼지집 |
| 레퍼런스 2 | 004 목구멍 |
| 선택 근거 | 업종(고기) 일치, 프리미엄 톤 일치, 섹션 10개 풀세트 |
```

### 블루프린트 로딩

선택된 레퍼런스의 `blueprint.json`을 읽는다.
경로: 레퍼런스 폴더 또는 하네스의 blueprints/ 폴더

---

## Phase 3: 블루프린트 매핑

### 매핑 규칙

블루프린트의 슬롯(slot)을 기획서(brief.md)의 값으로 교체한다:

| 블루프린트 슬롯 | 기획서 매핑 소스 |
|---------------|---------------|
| `[메인 슬로건]` | brief → 슬로건 |
| `[브랜드명]` | brief → 브랜드명 |
| `[메뉴1]`, `[메뉴2]`... | brief → 메뉴 테이블 |
| `[가맹비]`, `[교육비]`... | brief → 가맹 조건 테이블 |
| `[전화번호]` | brief → 대표 전화번호 |
| `[회사명]`, `[대표자]`... | brief → 회사 정보 |
| `[히어로 배경 이미지]` | brief → 준비된 자료의 히어로 이미지 |
| `[매장사진1]`... | brief → 준비된 자료의 매장 사진 |

### 색상 오버라이드

```
if 기획서에 brand_color가 있으면:
  blueprint.colors.accent = 기획서.brand_color
  나머지 colors는 레퍼런스 기본값 유지

if 기획서에 배경 선호가 있으면:
  밝은배경 → bg_primary=#FFFFFF, bg_secondary=#F5F5F5
  어두운배경 → 레퍼런스 기본값 유지
```

### 섹션 조정

기획서 내용에 따라 블루프린트의 섹션을 추가/제거:

```
if 성공사례가 [미입력] → success 섹션 제거
if 메뉴가 3개 미만 → menu 섹션에서 카드 수 조정
if 가맹비용이 모두 [미입력] → franchise 섹션에서 비용 테이블 제거, CTA만 유지
```

## Phase 4: 코드 생성

### landing-dev 스킬의 패턴으로 구현

1. 매핑 완료된 blueprint를 기반으로 index.html 생성
2. 각 섹션은 landing-dev의 HTML 구조 패턴을 따른다
3. 색상/폰트는 매핑된 blueprint.colors/fonts를 Tailwind config에 등록
4. JS는 landing-dev의 패턴 코드를 필요한 만큼만 포함

### 코드 생성 순서

```
1. <head> — Tailwind CDN + 폰트 CDN + tailwind.config
2. <nav> — 네비게이션 (로고 + 메뉴링크 + CTA 버튼)
3. 각 섹션 — blueprint.sections 순서대로
4. <footer> — 회사 정보
5. 플로팅 CTA — mobile 전용
6. <script> — JS 패턴 (스크롤, 카운팅 등)
```

---

## Phase 5: 사용자 자료 적용

### 이미지 연결

1. brief.md의 "준비된 자료"에서 파일 매핑 확인
2. 코드의 `images/파일명.jpg` 경로를 실제 파일명으로 교체
3. WebP 변환이 필요하면 안내 (수동 변환 필요)

### 텍스트 최종 확인

코드 생성 후 사용자에게:
- "슬로건이 `[현재 슬로건]`인데 맞나요?"
- "메뉴 설명을 확인해주세요"
- 수정이 필요한 부분을 바로 반영

### 미입력 항목 처리

```
[미입력] 상태인 텍스트 → 플레이스홀더로 표시
예: "여기에 슬로건을 입력하세요" (회색 텍스트)
사용자가 나중에 채울 수 있게 주석으로 위치 표시
```

## 체크리스트 (최종 검수)

코드 생성 완료 후 확인:

- [ ] 모든 `[미입력]` 슬롯이 실제 값으로 교체되었는가
- [ ] 이미지 경로가 실제 파일과 일치하는가
- [ ] 전화번호가 정확한가 (tel: 링크 포함)
- [ ] 모바일 반응형이 깨지지 않는가
- [ ] CTA 버튼이 #contact로 연결되는가
- [ ] 푸터의 회사 정보가 정확한가
- [ ] 플로팅 CTA가 모바일에서만 보이는가
